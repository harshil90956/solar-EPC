## Color Specifications

### Primary Colors
```javascript
Teal (Primary):    RGB(0, 102, 102)   // #006666
Gold (Accent):     RGB(255, 193, 7)   // #FFC107
Dark Gray:         RGB(51, 51, 51)    // #333333
Light Gray:        RGB(245, 245, 245) // #F5F5F5
```

### Usage Guidelines
- **Teal**: Headers, titles, accent bars, important text
- **Gold**: Highlights, bullet points, call-to-action
- **Gray**: Body text, secondary information
- **White**: Backgrounds, document badges

## Typography

| Element | Font | Size | Weight | Color |
|---------|------|------|--------|-------|
| Company Name | Helvetica | 18pt | Bold | White (on header) |

## Margins & Spacing

```
Page Margins:
- Left/Right: 15mm
- Top: 0mm (full bleed header)
- Bottom: 15mm

Section Spacing:
- Between sections: 8-10mm
- Within sections: 5mm
- Card padding: 5mm internal
```

## Comparison: Old vs New Format

### Traditional Multi-Page PDF
```
❌ 9+ pages (too long)
❌ Hard to quickly review
❌ Overwhelming details
❌ Print heavy
```

### New 2-Page Format
```
✅ Exactly 2 pages
✅ Quick overview
✅ Essential info only
✅ Print-friendly
✅ Professional appearance
✅ Easy to email
```

## Best Practices

### Content Guidelines
1. **Keep it concise** - Only essential information
2. **Use bullet points** - Easy to scan
3. **Highlight key numbers** - Total cost, savings
4. **Clear call-to-action** - Next steps

### Design Guidelines
1. **Consistent margins** - 15mm standard
2. **Color purpose** - Each color has meaning
3. **Proper hierarchy** - Headers > Subheaders > Body
4. **Whitespace** - Don't overcrowd

### Technical Guidelines
1. **A4 format** - Standard paper size
2. **Print-safe** - Works in B&W
3. **PDF/A compatible** - Archive quality
4. **Vector graphics** - Sharp at any zoom
